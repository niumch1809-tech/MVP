# AI 成本核验平台

面向制造业/硬件项目的 BOM 成本核验 MVP，用于多供应商 BOM 报价上传、解析、标准化展示、材料参考价核验和物料级成本对比。

当前阶段是 MVP 半自动版本，重点验证真实业务流程，不追求复杂 AI 或泛数据分析能力。

## 当前能力

- 上传 Excel / CSV 格式的供应商 BOM 或历史 BOM。
- 上传时填写产品名称和供应商名称，后续筛选、图表、表格和导出都会保留产品维度。
- 自动识别常见 BOM 字段并映射到标准字段。
- 支持宽表报价：同一 BOM 中多列供应商报价可拆成多供应商明细。
- 支持按产品、供应商、品类、物料名称搜索进行筛选。
- 供应商支持多选，便于 2-3 家或更多供应商并排对比。
- 报价对比图展示供应商总价、品类横坐标对比和物料级单价对比。
- 成本结构占比按供应商分别展示，避免把不同供应商混入同一个饼图。
- 数据表与预警支持网页内配置材料参考价源：
  - 输入价格接口 URL
  - 上传材料价格表 `.xlsx` / `.xls` / `.csv`
  - 无价格源时回退到 MVP mock 价格
- 明细表展示参考价、行情差异、行情风险和原始字段追溯。
- 数据输出支持完整成本对比表和明细数据 CSV 导出。

## 页面流程

1. `数据上传`: 上传 BOM/报价文件，填写产品名称、供应商名称和文件类型。
2. `报价对比图`: 查看供应商总价、品类对比、物料对比、分供应商成本结构占比。
3. `数据表与预警`: 查看 BOM 明细、数据异常、材料参考价偏离和来源追溯。
4. `数据输出`: 导出整合成本对比表或明细数据。

## 材料价格源

在 `数据表与预警` 中可以直接配置材料价格源，不需要 Codex 介入计算。

优先级：

1. 网页上传的材料价格表
2. 网页输入的价格接口 URL
3. 后端环境变量 `MATERIAL_PRICE_PROVIDER_URL`
4. MVP 内置 mock 价格源

材料价格表会尝试识别以下字段：

- `物料名称` / `材料名称` / `名称`
- `参考价` / `市场价` / `行情价` / `单价` / `价格`
- `品类`
- `单位`
- `币种`
- `来源`
- `更新时间`
- `备注`

价格接口 URL 需要支持 `POST`，请求体包含当前 BOM 行数据，并返回 `prices` 数组。

## 数据隐私

供应商 BOM、报价表、解析后的本地数据和材料价格表通常涉及商业隐私，默认不会提交到 GitHub。

当前 `.gitignore` 已忽略：

- `data/*.json`
- `uploads/*`
- `work/`
- `outputs/`
- `.env*.local`

仓库中只保留占位文件：

- `data/.gitkeep`
- `uploads/.gitkeep`

请不要把真实 BOM、报价、客户名、供应商名、材料价格表硬编码进源码或文档。需要样例时请使用脱敏 mock 数据。
