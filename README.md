# AI 成本核验平台

面向制造业/硬件项目的 BOM 成本核验 MVP，用于多供应商 BOM 报价上传、解析、标准化展示、材料参考价核验和物料级成本对比。

当前阶段是 MVP 半自动版本，重点验证真实业务流程，不追求复杂 AI 或泛数据分析能力。

## 当前能力

- 上传 Excel / CSV 格式的供应商 BOM 或历史 BOM。
- 上传时填写产品名称和供应商名称，后续筛选、图表、表格和导出都会保留产品维度。
- 自动识别常见 BOM 字段并映射到标准字段。
- 支持宽表报价：同一 BOM 中多列供应商报价可拆成多供应商明细。
- 支持按产品、供应商、品类、物料名称搜索进行筛选。
- 供应商支持多选，便于 2-3 家或更多供应商并排对比。
- 报价对比图展示供应商总价、品类横坐标对比和物料级单价对比。
- 成本结构占比按供应商分别展示，避免把不同供应商混入同一个饼图。
- 数据表与预警支持网页内配置材料参考价源：
  - 输入价格接口 URL
  - 上传材料价格表 `.xlsx` / `.xls` / `.csv`
  - 无价格源时回退到 MVP mock 价格
- 明细表展示参考价、行情差异、行情风险和原始字段追溯。
- 数据输出支持完整成本对比表和明细数据 CSV 导出。

## 页面流程

1. `数据上传`: 上传 BOM/报价文件，填写产品名称、供应商名称和文件类型。
2. `报价对比图`: 查看供应商总价、品类对比、物料对比、分供应商成本结构占比。
3. `数据表与预警`: 查看 BOM 明细、数据异常、材料参考价偏离和来源追溯。
4. `数据输出`: 导出整合成本对比表或明细数据。

## 材料价格源

在 `数据表与预警` 中可以直接配置材料价格源，不需要 Codex 介入计算。

优先级：

1. 网页上传的材料价格表
2. 网页输入的价格接口 URL
3. 后端环境变量 `MATERIAL_PRICE_PROVIDER_URL`
4. MVP 内置 mock 价格源

材料价格表会尝试识别以下字段：

- `物料名称` / `材料名称` / `名称`
- `参考价` / `市场价` / `行情价` / `单价` / `价格`
- `品类`
- `单位`
- `币种`
- `来源`
- `更新时间`
- `备注`

价格接口 URL 需要支持 `POST`，请求体包含当前 BOM 行数据，并返回 `prices` 数组。

## 数据隐私

供应商 BOM、报价表、解析后的本地数据和材料价格表通常涉及商业隐私，默认不会提交到 GitHub。

当前 `.gitignore` 已忽略：

- `data/*.json`
- `uploads/*`
- `work/`
- `outputs/`
- `.env*.local`

仓库中只保留占位文件：

- `data/.gitkeep`
- `uploads/.gitkeep`

请不要把真实 BOM、报价、客户名、供应商名、材料价格表硬编码进源码或文档。需要样例时请使用脱敏 mock 数据。

## 从 GitHub 下载并在本地运行

下面是给新用户或协作者的完整启动流程。

### 1. 准备环境

需要先安装：

- Git
- Node.js 20 或更高版本
- npm，通常会随 Node.js 一起安装

检查是否安装成功：

```bash
git --version
node -v
npm -v
```

如果 `node -v` 显示低于 20，建议先升级 Node.js。

### 2. 从 GitHub 下载项目

打开终端或 PowerShell，进入你想保存项目的目录，然后执行：

```bash
git clone https://github.com/niumch1809-tech/MVP.git
cd MVP
```

如果已经下载过项目，只需要进入项目目录并拉取最新代码：

```bash
cd MVP
git pull
```

### 3. 安装依赖

第一次运行项目时，需要安装依赖：

```bash
npm install
```

安装完成后，本地会生成 `node_modules/` 目录。这个目录不需要上传 GitHub。

### 4. 启动开发服务器

推荐开发/试用时使用：

```bash
npm run dev
```

看到类似下面的输出，表示启动成功：

```text
Local: http://localhost:3000
Ready
```

然后在浏览器打开：

```text
http://localhost:3000
```

注意：运行 `npm run dev` 的终端窗口不能关闭，关闭后网页服务也会停止。

### 5. Windows 一键启动

Windows 用户也可以直接双击项目根目录下的：

```text
start-local.cmd
```

它会自动进入项目目录并启动本地服务。弹出的命令窗口需要保持打开。

### 6. 生产模式本地运行

如果想模拟正式环境，可以先构建再启动：

```bash
npm run build
npm run start
```

然后访问：

```text
http://localhost:3000
```

### 7. 如何测试功能

1. 打开 `http://localhost:3000`。
2. 进入 `数据上传`。
3. 填写产品名称和供应商名称。
4. 上传 `.xlsx`、`.xls` 或 `.csv` 格式的 BOM/报价文件。
5. 上传后进入 `报价对比图` 查看总价、品类、物料图表。
6. 进入 `数据表与预警` 查看明细、异常和材料参考价偏离。
7. 进入 `数据输出` 导出整合成本对比表。

支持一个 Excel 文件内包含多个工作表。多个工作表会按工作表名称拆分为不同报价对象。

### 8. 本地数据保存位置

上传和解析后的数据保存在本地：

```text
data/bom-records.json
```

这些数据可能包含供应商报价和商业敏感信息，默认不会上传到 GitHub。`.gitignore` 已忽略 `data/*.json`。

如果想清空本地数据，可以在网页里点击清空，也可以删除本地的 `data/bom-records.json`。

### 9. 常见问题

如果浏览器打不开 `http://localhost:3000`：

1. 确认运行 `npm run dev` 或 `start-local.cmd` 的窗口还开着。
2. 确认终端里显示 `Ready`。
3. 如果提示端口被占用，可以关闭其他 Node/Next.js 进程，或重启电脑后重新运行。
4. 如果依赖报错，重新执行：

```bash
npm install
```

如果上传文件后没有数据：

1. 确认文件是 `.xlsx`、`.xls` 或 `.csv`。
2. 确认 BOM 表里有可识别的物料名称、数量、单价、金额等字段。
3. 如果文件里有多个工作表，系统会读取所有有效工作表。
4. 对于非常不规整的 BOM，可能仍需要继续增强字段映射和物料匹配规则。
